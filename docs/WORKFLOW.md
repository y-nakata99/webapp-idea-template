# WORKFLOW

このリポジトリでは、Pull Request ベースの開発フローを必須とする。

基本ルール：
・main ブランチは本番用ブランチとする
・main ブランチへ直接 commit しない
・すべての変更は Pull Request 経由で行う

標準開発手順：
1. main ブランチを最新化する
2. main から作業用ブランチを作成する
3. 実装を行う（既存ファイルは編集を優先）
4. 差分が明確な commit を作成する
5. main 向けの Pull Request を作成する
6. レビュー後、main にマージする
7. Vercel により本番デプロイされる

差分を見やすくするためのルール：
・既存ファイルを置き換えず、編集を優先する
・1 commit = 1 変更を意識する
・不要な整形のみの変更は行わない
