# 修正依頼（codex 用）

## 目的

差分が明確な Pull Request を作成すること。

## Git / ブランチ運用ルール
1. 作業開始は必ず main ブランチから行う
2. 作業ごとに main から新規ブランチを作成する
   ブランチ名：codex/<作業内容を表す短い名前>
3. main ブランチへ直接 commit しない
4. 既存ファイルは新規作成し直さず、編集を優先する
5. 差分が明確に確認できる形で commit する
6. 作業完了後は main 向けの Pull Request を作成する

## 実装方針
実装前に、どのように修正するかを簡潔に整理してから実装すること。
